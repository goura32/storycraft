# 次世代生成フロー設計

> 製品上の正本は [製品仕様](../product/SPECIFICATION.md) とする。この文書は、次世代実装における工程、正本、LLMの責務、検証境界を定める。

## 1. 目的

利用者は初回企画を一度だけ渡し、途中の確認なしに完結した複巻小説Markdownを得る。継続性のための情報は、本文の補助メモではなく、再開可能な正本台帳として保存する。

## 2. 正本と所有者

| 正本 | 所有する情報 | 変更できる工程 |
|---|---|---|
| 企画 | 利用者の題材、重要人物、希望、回避、結末、巻数 | 初回入力の正規化だけ |
| 全巻構成 | 巻の役割、変化、巻末の問い、結末条件 | 初期の全巻構成工程だけ |
| 人物台帳 | 固定プロフィール、開始時の目的・圧力・場所・状態・通常知識 | 現在状態だけを許可済み場面で更新 |
| 関係台帳 | 関係の意味、開始時の関係・読者知識 | 現在状態だけを許可済み場面で更新 |
| 世界台帳 | 固定事実、規則、利用条件、開始時の状態 | 現在状態だけを許可済み場面で更新 |
| 時間台帳 | 基準時点、期限、移動・回復の規則、開始状態 | 現在状態だけを許可済み場面で更新 |
| 主要項目台帳 | 作者の真実、開示規則、導入・回収条件、開始状態 | 導入・進展・回収状態だけを許可済み場面で更新 |
| 巻・章・場面設計 | 実行順、局所目的、許可された開示と状態遷移 | 対象の未執筆単位だけ |
| 場面成果物 | 凍結本文、本文根拠つき継続性要約、採用済み更新 | 採用前の対象場面だけ |

固定プロフィール、世界の規則、作者の真実、回収条件は、本文・要約・状態抽出のいずれも変更できない。

## 3. 工程

```text
企画の検証・正規化
→ 全巻構成
→ 人物
→ 関係
→ 世界
→ 時間
→ 主要項目
→ 初期台帳の横断確認・確定
→ [各巻: 章一覧 → 各章: 場面カード → 本文 → 継続性抽出]
→ 巻要約
→ 完結確認
→ Markdown出力
```

各LLM工程は次の共通ライフサイクルを使う。

```text
草稿 → 構造・参照検証 → 批評 → 必要時の修正 → 再検証 → 採用または草稿保持 → 保存
```

`run` は未完了工程を連続して実行する。`step` は一つの保存可能な工程単位を実行する。`resume` は保存済みの採用状態だけから未完了工程を再開する。

## 4. IDと参照

- 永続IDはプログラムが採番する。LLMは永続IDを新規作成、改名、再利用しない。
- 新規台帳の生成は内容だけを返す。コードは配列順でIDを採番してから保存する。
- 関係、世界、時間、主要項目は、すでに保存・採番された候補カタログのIDだけを参照する。
- 場面カードは、対象場面に必要な候補カタログだけを受け、可視IDと更新許可IDを返す。更新許可IDは可視IDの部分集合でなければならない。
- 未知ID、重複ID、未提示ID、固定情報への更新は採用しない。

## 5. LLMの役割分担

| 工程 | LLMの役割 | 出力 | 禁止事項 |
|---|---|---|---|
| 全巻構成 | シリーズ設計 | 巻の役割・変化・問い・結末条件 | 台帳の詳細、章・場面本文の確定 |
| 初期台帳 | 設定設計 | 新規台帳内容と開始状態 | 永続IDの決定、他台帳の固定情報の上書き |
| 巻・章設計 | 構成設計 | 章の目的、開始・終了状態、場面数 | 将来巻の詳細、本文の執筆 |
| 場面カード | 局所設計 | POV、場所、必須行動、開示範囲、許可更新 | 未提示の主要台帳項目、新規主要伏線 |
| 本文 | 小説執筆 | `content` だけ | 状態更新、要約、本文外事実、固定情報変更 |
| 継続性抽出 | 凍結本文の事実抽出 | 根拠つき要約と実際の更新 | 本文にない事実、未許可更新、本文修正 |
| 巻要約 | 巻本文の引継ぎ要約 | 次巻のための本文根拠つき要約 | 台帳・本文の上書き |
| 完結監査 | 未回収の発見 | 根拠つき監査結果 | 完結の自己承認 |

## 6. 本文と状態抽出の分離

本文執筆の応答は完成本文だけとする。本文を凍結した後、継続性抽出工程が場面カードに許可された更新候補を本文と照合する。

各更新は、対象ID、変更対象の現在状態、変更後の値、本文中の根拠位置を持つ。コードは対象ID、可視性、許可、変更可能なフィールド、本文根拠位置を検証してから台帳に適用する。根拠のない更新は採用しない。

## 7. 完結判定

完結はLLMの自己申告では決めない。コードが以下を検証する。

1. 登録済みの主要項目がすべて回収済みである。
2. 最終巻の結末条件を満たす本文根拠がある。
3. 計画された巻・章・場面に空本文や欠落がない。
4. 巻本文に重複がない。

LLMの完結監査は、上記を満たさない理由や根拠不足を検出する補助である。
