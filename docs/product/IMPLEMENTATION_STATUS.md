# 現行実装の位置付け

> この文書は、現行コードの成熟度を短く示す。製品仕様ではない。

現行コードは、Storycraft の次世代仕様を検証するための参照実装である。破壊的な再設計を予定しているため、現行の内部構造、コマンド引数、保存形式、プロンプト構成を将来の互換契約として扱わない。

## 仕様との関係

- [製品仕様](SPECIFICATION.md)が次の正式リリースの正本である。
- 現行コードが製品仕様と異なる場合、製品仕様を現行コードに合わせて弱めない。
- 実装を改修するときは、製品仕様の契約を満たすことを優先する。

## 現在の扱い

- 現行コードは設計検証・実験用途に限る。
- 完走性、出力品質、操作契約は次世代実装であらためて検証する。
- 具体的な未対応事項や改修計画は、実装作業の課題・テスト・変更履歴で管理する。この文書へ一時的な進捗を蓄積しない。
