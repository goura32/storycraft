# 現行実装の位置付け

> この文書は、現行コードの成熟度を短く示す。製品仕様ではない。

現行コードは、Storycraft の現行仕様に基づく互換なしの実装である。旧実装の内部構造、コマンド引数、保存形式、プロンプト構成は継承しない。公開操作は `SeriesService`、内部責務は契約検証・状態保存・出力・ワークフローへ分離されている。

## 仕様との関係

- [製品仕様](SPECIFICATION.md)が実装の正本である。
- 実装が製品仕様と異なる場合、製品仕様を実装に合わせて弱めない。
- 実装を改修するときは、製品仕様の契約を満たすことを優先する。

## 現在の扱い

- 利用者操作は `run`、`resume`、`step` を対象とする。新規作業場所は手入力briefまたは自由keywordsのどちらか一方から開始し、keywords時はLLM生成briefを手入力briefと同じ構造契約で採用する。
- Canon確定後に既存thread IDを `volume_map` へ配分し、後続の章・場面設計は対象巻の配分を入力として扱う。
- 作業場所は単一シリーズ用であり、公開操作は排他lockで保護する。
- stateと完成出力は、同期と原子的置換により途中の成果物を公開しない。
- 実LLMによる作品の内容品質や販売原稿としての品質は、別途の実行監査まで主張しない。
- 具体的な未対応事項や改修計画は、実装作業の課題・テスト・変更履歴で管理する。この文書へ一時的な進捗を蓄積しない。
