# 現行実装の位置付け

> この文書は、現行コードの成熟度を短く示す。製品仕様ではない。

現行コードは、Storycraft の次世代仕様に基づく互換なしの実装である。旧実装の内部構造、コマンド引数、保存形式、プロンプト構成は継承しない。

## 仕様との関係

- [製品仕様](SPECIFICATION.md)が実装の正本である。
- 実装が製品仕様と異なる場合、製品仕様を実装に合わせて弱めない。
- 実装を改修するときは、製品仕様の契約を満たすことを優先する。

## 現在の扱い

- 利用者操作は `run`、`resume`、`step` を対象とする。
- 正本状態、場面採用時の更新、再開、Markdown出力は次世代形式だけを扱う。
- 実LLMによる作品の内容品質や販売原稿としての品質は、別途の実行監査まで主張しない。
- 具体的な未対応事項や改修計画は、実装作業の課題・テスト・変更履歴で管理する。この文書へ一時的な進捗を蓄積しない。
