# Storycraft 仕様

> この文書は、現在の実装方針を記載する。要件を守る範囲で変更してよい。
> 変えてはいけない製品上の約束は[要件](REQUIREMENTS.md)を参照する。

## 1. 初回入力

利用者は最初に次を入力する。

- 仮題
- ジャンルと雰囲気
- 主人公
- 重要人物
- 起きてほしいこと
- 避けたいこと
- 最後にどう終えたいか
- 巻数。省略時は5巻

巻数は4〜10の範囲だけを受け付ける。

## 2. 自動実行

1. 全巻の計画を作る。
2. 各巻の目的、人物関係の変化、巻末の引きを決める。
3. 巻を章と場面へ分ける。
4. 場面を順番に書く。
5. 各場面から、次の場面に必要な短い引継ぎ要約を作る。
6. 巻末で巻の本文をまとめる。
7. 最終巻の後に全巻をまとめる。

通常の実行では、途中確認を求めない。利用者が明示的に停止した場合だけ中断する。

## 3. 生成失敗時

本文が空、必要な見出しが欠ける、または形式として読めない場合は、その単位だけ再試行する。

表現の好み、軽い矛盾、細かな文章品質は停止条件にしない。必要なら一度だけ書き直し、それでも残る問題は記録して先へ進む。

## 4. 出力

出力先の初期形は次とする。

```text
output/
  volume-01.md
  volume-02.md
  ...
  volume-10.md
  series.md
```

実際に作る巻数に対応するファイルだけを出力する。

各巻には巻題、章題、本文を含める。`series.md`には全巻を順番に収録する。

## 5. KDP向けの扱い

初巻を無料導入巻として扱うが、KDPでの無料配布の実現方法は、公開時点のKDP規約と販促制度に従う。

初期版はKDPへの自動入稿を行わない。Markdown出力と、巻数・巻題・販売上の役割をまとめた出版用メモを作る。

## 6. 保存する情報

実装では、次の最小限だけを保存する。

- 初回企画
- 全巻計画
- 各場面の本文
- 各場面の引継ぎ要約
- 実行位置

同じ設定を複数形式で複写しない。停止後の再開では、保存済み本文と引継ぎ要約を利用する。
