# Storycraft 要件

> この文書は、製品として必ず満たす利用者向けの約束だけを記載する。実装方法、保存形式、LLM呼び出し名、再試行回数は[製品仕様](SPECIFICATION.md)で扱う。

## 1. 目的

Storycraftは、日本語で個人出版・販売できる、読者が最後まで読み進められる完結小説シリーズを、LLMで自動作成する道具である。

売上は保証しない。ただし、単発の断片ではなく、続巻を読みたくなるシリーズを作ることで収益化を目指す。

## 2. 利用者の関わり方

- 利用者はシリーズ開始時に、手入力briefまたは自由なkeywordsのどちらか一方を一度入力する。keywordsにはジャンル、巻数、イメージ、題材、人物像、避けたい要素などを混在できる。
- 通常の実行では、途中の巻・章・場面ごとの確認や介入を求めない。道具はシリーズ初期設計から完結監査、出力までを自動で行う。
- LLMレビューは品質改善に使う。レビューのseverity、合否、継続可能性だけで処理を停止しない。
- 実行が中断しても、利用者は初回入力を再入力せずに同じシリーズを再開できる。

## 3. シリーズの巻数

- シリーズは4〜10巻で構成する。
- 5巻を推奨値とする。
- 最終巻は、開始時に決めた結末条件を満たして完結する。
- 各巻は一冊として読める区切りを持ち、最終巻以外は次巻を読みたくなる問いを残す。

## 4. 販売の前提

- 初巻は、読者がシリーズへ入るための無料導入巻として扱う。
- 2巻目以降は、個人出版で販売する前提とする。
- 販売先は最初はKDPを想定する。
- 無料配布の具体的方法は販売先の規約に従う。初巻無料という販売方針自体は維持する。
- 道具はKDPへの自動入稿・自動公開を行わない。

## 5. 最終出力

- 各巻の完成原稿をMarkdownで出力する。
- 全巻を順番にまとめたMarkdownを出力する。
- 出力だけで、販売用の原稿整形へ進められる状態にする。
- LLMレビューで残った品質issueは監査記録として残すが、それだけで機械的な完成条件を満たす出力を隠さない。

## 6. 物語として守ること

- 主人公、重要人物、物語の核、結末は開始時のbriefを基にする。keywords時は、LLMが生成して構造契約を通過したbriefを開始時のbriefとする。
- 本文前に、人物、関係、世界、時間規則、主要な問い・伏線・重要イベント、結末条件を含むシリーズ初期設計を確定する。
- 各巻は、人物または中心関係を変化させ、主要な問い・伏線・重要イベントを導入、進展、回収する役割を持つ。
- 本文で根拠が明示された新しい局所Canon項目は、既存項目との重複を避けて追加できる。
- 最終巻で、開始時のbriefに示した主要な問いと結末を回収する。
- 空の本文、同じ本文の重複、または必要な章・場面の欠落がある巻は完成扱いにしない。
- 表現の好み、文字数の目安からのずれ、LLMによる軽い不整合指摘は、全シリーズの完成を妨げない。
