# Storycraft 要件

> この文書は、製品として必ず満たす約束だけを記載する。
> 実装方法、画面、保存形式、LLM、再試行回数、販売先ごとの手続きは記載しない。それらは[仕様](SPECIFICATION.md)で扱う。

## 1. 目的

Storycraftは、日本語で個人出版・販売できる、読者が最後まで読み進められる完結小説シリーズを、LLMで作る道具である。

売上は保証しない。ただし、単発の断片ではなく、続巻を読みたくなるシリーズを作ることで収益化を目指す。

## 2. 利用者の関わり方

利用者は、シリーズ開始時に企画を一度入力する。

通常の実行では、その後の巻・章・場面ごとの確認や介入を求めない。道具は全巻の計画、執筆、最終出力までを自動で行う。

実行が中断しても、利用者は初回企画を再入力せずに同じシリーズを再開できる。

## 3. シリーズの巻数

- シリーズは4〜10巻で構成する。
- 標準の巻数は5巻とする。
- 最終巻は、開始時に決めた結末を回収して完結する。
- 各巻は、仕様で定める販売用原稿として十分な分量を持つ。
- 各巻は一冊として読める区切りを持ち、最終巻以外は次巻を読みたくなる問いを残す。

## 4. 販売の前提

- 初巻は、読者がシリーズへ入るための無料導入巻として扱う。
- 2巻目以降は、個人出版で販売する前提とする。
- 販売先は最初はKDPを想定する。
- 無料配布の具体的方法は販売先の規約に従う。初巻無料という販売方針自体は維持する。
- 道具はKDPへの自動入稿・自動公開を行わない。

## 5. 最終出力

- 各巻の完成原稿をMarkdownで出力する。
- 全巻を順番にまとめたMarkdownを出力する。
- 出力だけで、販売用の原稿整形へ進められる状態にする。

## 6. 物語として守ること

- 主人公、重要人物、物語の核、結末は開始時の企画を基にする。
- 各巻で人物関係または物語上の問題が変化する。
- 最終巻で、開始時に示した主要な問いと結末を回収する。
- 空の本文、同じ本文の重複、または必要な章の欠落がある巻は完成扱いにしない。
- 表現の好みや軽い不整合は、全シリーズの完成を妨げない。
